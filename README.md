# 🏢 统一协作办公系统（OA System）
 
> 基于 Django + DRF 开发的企业级协作办公后端系统，支持考勤管理、通知公告、员工管理等核心功能，采用 JWT 鉴权与部门级权限控制。
 
## ✨ 功能模块
 
- 📊 **首页数据看板**：可视化展示员工考勤情况、部门统计等数据图表
- 📅 **考勤管理**：支持考勤发布与处理，管理员可查看全员考勤记录
- 📢 **通知公告**：支持发布通知、处理通知、阅读量统计、未读消息提醒（小红点）
- 👁️ **通知可见范围**：支持设置全部门可见或指定部门可见
- 👥 **员工管理**：支持增删员工，员工账号通过邮件激活（服务端发送激活链接）
- 🔐 **权限控制**：基于 JWT + 部门角色实现接口级权限管理，不同部门拥有不同操作权限
## 🛠️ 技术栈
 
- **框架**：Django、Django REST Framework（DRF）
- **数据库**：MySQL
- **缓存**：Redis（缓存热点数据，降低数据库压力）
- **异步任务**：Celery + Redis（处理邮件发送、通知推送等耗时任务）
- **鉴权**：JWT（JSON Web Token）
- **部署**：Docker 容器化部署
## 🚀 快速开始
 
**环境要求**
```
Python 3.8+
MySQL 5.7+
Redis 6.0+
```
 
**安装依赖**
```bash
pip install -r requirements.txt
```
 
**配置环境变量**
```bash
# 复制环境变量模板
cp .env.example .env
# 填写数据库、Redis、邮箱等配置
```
 
**数据库迁移**
```bash
python manage.py migrate
```
 
**启动 Celery 异步任务**
```bash
celery -A 项目名 worker -l info
```
 
**启动服务**
```bash
python manage.py runserver
```
 
## 📌 项目亮点
 
- JWT + 部门角色双重鉴权，实现精细化接口级权限控制
- Celery + Redis 异步处理员工激活邮件发送，避免请求阻塞
- Redis 缓存热点数据（组织架构、通知列表），显著降低数据库 I/O 压力
- 通知系统支持未读提醒、阅读量统计、可见范围设置等完整功能
- Docker 容器化部署，确保开发与生产环境一致性
